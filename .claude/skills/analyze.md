# /analyze

Прочитать модель через MCP-сервер `finmodel` и дать аналитическое заключение.

**Никаких Excel-чтений.** Сервер сам знает горизонт, фазы и итоги. Два MCP-вызова.

## Синтаксис

```
/analyze <name>
```

---

## Шаг 1 — Получить KPI и итоги

```
mcp__finmodel__get_kpi
  project_name = "<name>"
```

Возвращает: `npv`, `irr_annual`, `roi`, `payback_months` + суммарные значения за горизонт
(`total_net_profit`, `total_revenue_net`, `total_income_tax`, `total_vat_paid`,
`total_vat_refund`, `total_property_tax`, `total_interest_paid`,
`total_revenue_gross`, `total_operating_costs`).

## Шаг 2 — Получить начало и конец CF

```
mcp__finmodel__get_cashflow_summary
  project_name = "<name>"
  n_first      = 12
  n_last       = 12
```

Возвращает:
- `horizon_months`, `construction_months` — для масштаба
- `first` — первые 12 строк CF (фаза строительства + выход на эксплуатацию)
- `last` — последние 12 строк (зрелая операционка)

Каждая строка: `month`, `date`, `phase`, `revenue_net`, `ebit`, `interest`,
`net_profit`, `investor_cf`, `cumulative_cf`, `loan_balance`.

> **Если нужны строки вокруг точки окупаемости** (`payback_months` из шага 1) —
> можно вызвать `get_cashflow_summary` с `n_first = payback_months + 3`, тогда
> хвост `first` покажет переход CF через ноль.

## Шаг 3 — Получить параметры (для контекста)

```
mcp__finmodel__get_project_params
  project_name = "<name>"
```

Нужны для пояснений: «при ставке ЦБ 14,5% и аренде 1100 руб./кв.м. модель показывает …».

---

## Шаг 4 — Сформировать заключение

Структура:

```markdown
## Аналитическое заключение: <name>

### Инвестиционная привлекательность
[NPV > 0 → создаёт стоимость; IRR vs ставка ЦБ; общий вывод 1–2 предложения]

### Ключевые показатели
| Показатель | Значение | Оценка |
|---|---|---|
| NPV | <npv> руб. | положительный / отрицательный |
| IRR | <irr_annual>% | выше / ниже ставки ЦБ (<key_rate>%) |
| ROI | <roi>% | высокий / средний / низкий |
| Срок окупаемости | <payback_months> мес. | в пределах горизонта <horizon_months> / нет |

### Структура CF (по шагу 2)
[Краткое описание: первые <construction_months> месяцев — отток на стройку;
выход на положительный месячный CF в мес. ~N; накопленный CF на конец горизонта =
last[-1].cumulative_cf руб.]

### Финансовая нагрузка
- Доля налогов в выручке: (total_income_tax + total_vat_paid + total_property_tax) / total_revenue_net = X%
- Проценты vs чистая прибыль: total_interest_paid / total_net_profit = Y%
- Долговая нагрузка на пике: max(loan_balance) ≈ Z руб. (берётся из first[-1] или известна по структуре)

### Риски
[Топ-3 риска на основе чисел. Примеры:
 — IRR близок к ставке ЦБ → чувствительность к её росту
 — Срок окупаемости > 50% горизонта → длинный цикл
 — Высокая доля кредита → нагрузка процентов]

### Рекомендации
[Конкретные действия: пересмотреть структуру финансирования / поторговаться
по ставке кредита / индексировать аренду чаще / сократить срок строительства]

### Вывод
[1–2 предложения: целесообразность инвестиции]
```

---

## Критерии оценки

| KPI | Хорошо | Средне | Плохо |
|---|---|---|---|
| NPV | > 0 | ≈ 0 | < 0 |
| IRR | > ставки ЦБ + 5 п.п. | в диапазоне ЦБ ± 5 п.п. | < ставки ЦБ |
| Срок окупаемости | < 50% горизонта | 50–75% | > 75% или нет в горизонте |
| ROI | > 100% | 30–100% | < 30% |

## Стиль

- Числа: `48 800 000 руб.`, ставки `15,0%`, периоды `120 мес. (10 лет)`.
- Не цитировать названия ячеек Excel в выводе пользователю.
- Говорить языком бизнеса: «модель создаёт стоимость», а не «NPV положительный».
