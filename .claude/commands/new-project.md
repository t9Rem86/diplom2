# /new-project

Собрать параметры проекта склада в диалоге и сохранить в `projects/<name>.json`.
Файл потом читает `/generate-report`.

**Никаких Python-скриптов.** JSON пишется напрямую через инструмент Write.

## Диалог с пользователем

Задавай вопросы поочередно. После получения всех ответов — показать сводку, подтвердить, сохранить JSON.

### Блок 1 — Объект
1. **Название проекта** (латиницей или кириллицей, например: `Sklad_A1`, `Склад Челябинск`)
2. **Площадь склада** (кв.м.)
3. **Дата начала строительства** — **обязательно спросить у пользователя.**
   Формат: ДД.ММ.ГГГГ, например: `01.07.2026`.
   Никаких дефолтов, никаких «сегодня + N месяцев». Если пользователь даёт неверный формат —
   переспросить, привести к ДД.ММ.ГГГГ. Эта дата запишется в `start_date` JSON и попадёт
   в шапку Excel-отчёта.
4. **Срок полезного использования объекта (СПИ)** — **обязательно спросить у пользователя.**
   В годах. Влияет на амортизацию и налог на имущество — критичный параметр для модели.
   Ориентиры: склад класса A металлоконструкция → 20–30 лет, монолит → 30–40 лет,
   реконструкция/б/у → 10–15 лет. Если пользователь не уверен — предложить **20 лет**
   как типовое значение для современного складского комплекса (но не подставлять молча).

### Блок 2 — Стоимость и финансирование
4. **Стоимость проекта с НДС** (руб.)
   - Ориентир: 40 000–90 000 руб./кв.м. для склада класса A/B
5. **Доля собственных средств** (0–1, например `0.20` = 20%)
6. **Ключевая ставка ЦБ** (например `0.15` = 15%)
7. **Ставка по кредиту** (например `0.20` = 20% ≈ ЦБ + 5 п.п.)

### Блок 3 — Аренда
8. **Арендная ставка без НДС** (руб./кв.м./мес.)
   - Ориентир: 800–1 500 руб./кв.м./мес. для склада класса A
9. **Индексация аренды** (год., например `0.05` = 5%)

### Блок 4 — Сроки и налоги
10. **Срок строительства** (мес., обычно 4–12)
11. **Горизонт планирования** (мес., обычно 60–120)
12. **НДС** (по умолчанию `0.22`)
13. **Налог на прибыль** (по умолчанию `0.25`)
14. **Налог на имущество** (год., по умолчанию `0.022`)
15. **OPEX** (доля от выручки, по умолчанию `0.10`)

```

   Объект:         <area_sqm> кв.м., начало <start_date>, СПИ <useful_life_years> лет
   Стоимость:      <cost_with_vat> руб. с НДС
   Финансирование: СС <equity_share×100>% / Кредит <(1-equity_share)×100>%
   Ставки:         ЦБ <key_rate×100>% / Кредит <loan_rate×100>%
   Аренда:         <rent_rate_ex_vat> руб./кв.м./мес. (без НДС)
   Индексация:     <rent_indexation×100>%/год
   Строительство:  <construction_months> мес.
   Горизонт:       <horizon_months> мес.

Сохраняю projects/<name>.json...
```

## Сохранить JSON через Write

Инструмент `Write`, путь `projects/<name>.json`, содержимое:

```json
{
  "name": "<name>",
  "cost_with_vat": <число>,
  "area_sqm": <число>,
  "equity_share": <число>,
  "rent_rate_ex_vat": <число>,
  "rent_indexation": <число>,
  "key_rate": <число>,
  "loan_rate": <число>,
  "vat_rate": 0.22,
  "horizon_months": <число>,
  "construction_months": <число>,
  "income_tax": 0.25,
  "property_tax": 0.022,
  "opex_pct": 0.10,
  "useful_life_years": <число лет, спросить у пользователя>,
  "start_date": "<ДД.ММ.ГГГГ>"
}
```

Имена полей в JSON **обязаны** совпадать с полями из существующих файлов
(`sklad_vesna.json`, `sklad_chelyabinsk.json`), потому что `/generate-report`
читает именно эти ключи.

## После сохранения

```
✅ Проект сохранён: projects/<name>.json

Следующий шаг: /generate-report <name>  — построить Excel-модель через MCP
```
