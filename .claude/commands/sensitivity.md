# /sensitivity

> **Обычно не нужна.** Лист «Чувствительность» **автоматически** заполняется при `/generate-report <name>`
> прямо во время сборки Excel (через `build_sensitivity` в `build_excel.py`). Базовый набор:
> ±20% по себестоимости, аренде, кредитной ставке, инфляции и ставке ЦБ. Все значения —
> формулы-ссылки на «Параметры», поэтому при изменении входных данных сценарии сами пересчитываются.
>
> Эта команда нужна только если хочешь **перезаписать** лист другими сценариями
> через MCP (например, ±30%, ±50% или иной набор параметров) — без пересборки модели.

Добавить сценарный анализ чувствительности на лист «Чувствительность» через MCP.
**Никаких Python-скриптов.** Сценарные значения параметров — формулы-ссылки на «Параметры».

## Синтаксис

```
/sensitivity <name>
```

---

## Шаг 1 — Прочитать базовые параметры

```
mcp__excel__read_data_from_excel
  filepath   = "reports/<name>_finmodel.xlsx"
  sheet_name = "Параметры"
  start_cell = "A1"
  end_cell   = "C28"
```

Используется только для контроля. Все значения в сценарной таблице записываются как формулы-ссылки на «Параметры», поэтому числа из шага 1 в сами ячейки не подставляются.

---

## Шаг 2 — Записать таблицу сценариев на лист «Чувствительность»

Один вызов `write_data_to_excel` с `data: List[List]`. Каждая ячейка-значение — это формула-ссылка `='Параметры'!$C$N*коэф`.

```
mcp__excel__write_data_to_excel
  filepath   = "reports/<name>_finmodel.xlsx"
  sheet_name = "Чувствительность"
  start_cell = "A1"
  data = [
    ["АНАЛИЗ ЧУВСТВИТЕЛЬНОСТИ: <name>", "", "", "", ""],
    ["", "", "", "", ""],
    ["Параметр", "Пессимист (−20%)", "Базовый", "Оптимист (+20%)", "Влияние"],
    ["Себестоимость строительства, руб.",
       "='Параметры'!$C$5*1.2",
       "='Параметры'!$C$5",
       "='Параметры'!$C$5*0.8",
       "Рост затрат → ↓ NPV"],
    ["Арендная ставка, руб./кв.м./мес.",
       "='Параметры'!$C$7*0.8",
       "='Параметры'!$C$7",
       "='Параметры'!$C$7*1.2",
       "Снижение ставки → ↓ выручки"],
    ["Кредитная ставка",
       "='Параметры'!$C$10*1.2",
       "='Параметры'!$C$10",
       "='Параметры'!$C$10*0.8",
       "Рост ставки → ↑ процентов"],
    ["Инфляция",
       "='Параметры'!$C$8*0.8",
       "='Параметры'!$C$8",
       "='Параметры'!$C$8*1.2",
       "Рост инфляции → ↑ выручки"],
    ["Ставка ЦБ",
       "='Параметры'!$C$9*1.2",
       "='Параметры'!$C$9",
       "='Параметры'!$C$9*0.8",
       "Рост ставки → ↓ NPV"],
    ["", "", "", "", ""],
    ["БАЗОВЫЕ KPI (ссылки на лист ИТОГ)", "", "", "", ""],
    ["NPV", "руб.", "='ИТОГ'!$C$7", "", ""],
    ["IRR", "%",   "='ИТОГ'!$C$8", "", ""],
    ["ROI", "%",   "='ИТОГ'!$C$9", "", ""],
    ["Срок окупаемости", "мес.", "='ИТОГ'!$C$10", "", ""],
    ["", "", "", "", ""],
    ["ИНСТРУКЦИЯ", "", "", "", ""],
    ["1. Откройте лист «Параметры»", "", "", "", ""],
    ["2. Измените нужный параметр (C4:C17)", "", "", "", ""],
    ["3. Excel автоматически пересчитает все листы", "", "", "", ""],
    ["4. Прочитайте новые NPV/IRR/ROI на листе «ИТОГ»", "", "", "", ""],
    ["", "", "", "", ""],
    ["Для автоматической таблицы сценариев:", "", "", "", ""],
    ["Excel → Данные → Анализ «что если» → Таблица данных", "", "", "", ""]
  ]
```

> **Правило MCP:** все подсписки одинаковой длины (5 элементов). Пустые ячейки = `""`, а не `None`.
> Номера строк KPI на листе ИТОГ: NPV=C7, IRR=C8, ROI=C9, Payback=C10 (по структуре, заданной в /use-template).

## Шаг 3 — Аналитическое резюме

После записи сформулировать качественный вывод на основе структуры модели:

```
✅ Анализ чувствительности добавлен: reports/<name>_finmodel.xlsx → «Чувствительность»

Ключевые факторы риска для проекта <name>:

  1. Арендная ставка — прямо влияет на 100% выручки. Снижение на 20% → выручка −20%.
  2. Себестоимость строительства — влияет на CAPEX, кредит, амортизацию, НДС.
     Рост на 20% → кредит ×1,2, ежемесячные проценты ×1,2.
  3. Кредитная ставка — влияет на ежемесячные проценты в фазе эксплуатации.
  4. Инфляция — положительный фактор (рост аренды), но ограниченный первым годом.
  5. Ставка ЦБ — только дисконтирование (NPV), не влияет на денежные потоки.

Для количественной оценки NPV каждого сценария → измените параметр на листе «Параметры»
и прочитайте NPV на листе «ИТОГ».
```
