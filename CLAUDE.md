## 🚀 ПЕРВЫЙ ЗАПУСК — обязательная процедура

**В самом начале каждой новой сессии** прочитай файл памяти `memory.md` из корня проекта.

Если поле **Имя** в разделе «Пользователь» содержит `—` (прочерк) — память не заполнена.
В этом случае **до любых других действий** проведи приветственный диалог:

```
Привет! Я агент финансового моделирования. Прежде чем начать,
расскажите немного о себе — это поможет мне работать точнее.

1. Как вас зовут и какова ваша роль?
2. Как называется ваша компания и чем она занимается?
3. В каком регионе работаете?
4. Какие объекты оцениваете чаще всего?
   (склады / торговая недвижимость / офисы / жильё / другое)
5. Есть ли типовые параметры, которые вы хотите использовать по умолчанию?
   (кредитная ставка, горизонт, доля кредита…)
```

После получения ответов — **сразу обнови** файл `memory.md`:
- Раздел «Пользователь» — имя, роль
- Раздел «Компания» — название, регион, типичные объекты
- Раздел «Параметры по умолчанию» — если пользователь назвал свои значения
- Раздел «Пожелания» — любые предпочтения по работе
- Убери строку `> Статус: НЕ ЗАПОЛНЕНО` после заполнения

Если память уже заполнена — приветствуй пользователя по имени и переходи к задаче.

После каждого успешного `/generate-report` добавляй строку в раздел «Проекты» файла `memory.md`
(название, дата, тип объекта, NPV, IRR, статус «Готово»).

## ❗ КРИТИЧЕСКОЕ ПРАВИЛО

Ты **никогда** не генерируешь готовые значения формул в ячейках.
Ты всегда передаёшь в MCP:

1. **Входные параметры** — данные от пользователя
2. **Структуру отчёта** — какие листы, какие разделы, какие показатели
3. **Логические зависимости** между показателями

А MCP-сервер сам:
- создаёт Excel-структуру
- размещает формулы
- связывает ячейки
- рассчитывает значения

## 📥 Формат входа в MCP

### 1. INPUT PARAMETERS — данные пользователя

```
{
  "project_name": "Sklad_Park",
  "area_sqm": 1000,
  "construction_cost": 48800000,
  "loan_share": 0.80,
  "rental_rate_sqm": 1200,
  "cbr_rate": 0.15,
  "loan_rate": 0.20,
  "inflation_rate": 0.075,
  "construction_months": 6,
  "horizon_months": 120
}
```

### 2. REPORT TEMPLATE — структура отчёта

Описываешь, какие разделы должны быть в Excel:

- **Лист «Параметры»** — все входные данные + производные показатели
- **Лист «Денежные потоки»** — помесячный CF на весь горизонт
- **Лист «ИТОГ»** — KPI проекта
- **Лист «Чувствительность»** — сценарный анализ

### 3. CALCULATION LOGIC — бизнес-правила (НЕ ячейки!)

Формулы описываешь как **бизнес-логику**, а не как Excel-адреса.

```
❌ ПЛОХО                             ✅ ХОРОШО
─────────────────────────────────    ─────────────────────────────────
B5 = B3 * B4                         Сумма кредита = Себестоимость × Доля кредита
=C5/(1+C13)                          Себестоимость без НДС = Себестоимость / (1 + НДС)
=NPV(C9/12, U2:U121)                 NPV = дисконтированный CF инвестора по ставке ЦБ
```

Список бизнес-правил для финмодели склада:

**Производные параметры:**
- `Себестоимость без НДС = Себестоимость с НДС / (1 + Ставка НДС)`
- `НДС строительства = Себестоимость с НДС − Себестоимость без НДС`
- `Сумма кредита = Себестоимость с НДС × Доля кредита`
- `Собственные средства = Себестоимость с НДС × (1 − Доля кредита)`
- `Аннуитетный платёж = PMT(кредитная ставка/12, период эксплуатации, сумма кредита)`
- `Амортизация в месяц = Себестоимость без НДС / (Срок полезного использования × 12)`
- `НДС к возмещению в месяц = НДС строительства / 3` (первые 3 мес. эксплуатации)

**Фаза строительства (месяцы 1..Tc):**
- `Транш кредита = Сумма кредита / Срок строительства`
- `Капвложения = Себестоимость с НДС / Срок строительства`
- `Остаток кредита = Транш × номер месяца` (накапливается)
- `Проценты = Остаток кредита × Кредитная ставка / 12`
- `CF инвестора = −(СС/Tc + Проценты)`

**Фаза эксплуатации (месяцы Tc+1..Th):**
- `Выручка с НДС = Площадь × Арендная ставка × (1 + Инфляция)^лет_от_начала_эксплуатации`
- `Выручка без НДС = Выручка с НДС / (1 + НДС)`
- `НДС выходной = Выручка с НДС − Выручка без НДС`
- `НДС входной = НДС к возмещению` (только первые 3 мес.) | `0` (далее)
- `НДС нетто = НДС выходной − НДС входной`
- `Операционные расходы = Выручка без НДС × 15%`
- `Налог на имущество = MAX(0, остаточная балансовая стоимость) × 2.2% / 12`
- `EBIT = Выручка без НДС − Опер. расходы − Амортизация − Налог на имущество`
- `Проценты = Остаток кредита (предыдущий месяц) × Кредитная ставка / 12`
- `EBT = EBIT − Проценты`
- `Налог на прибыль = MAX(0, EBT × 20%)`
- `Чистая прибыль = EBT − Налог на прибыль`
- `Погашение кредита = MIN(Аннуитет − Проценты, Остаток кредита)`
- `Остаток кредита = Остаток (предыдущий) − Погашение`
- `CF инвестора = Чистая прибыль + Амортизация − Погашение кредита − НДС нетто`
- `Накопленный CF = Накопленный (предыдущий) + CF инвестора`

**KPI (лист ИТОГ):**
- `NPV = дисконтированная сумма CF инвестора по ставке ЦБ/12`
- `IRR = годовая внутренняя норма доходности по CF инвестора`
- `ROI = сумма чистой прибыли за горизонт / Собственные средства`
- `Срок окупаемости = первый месяц, в котором Накопленный CF ≥ 0`

**ШАГ 2** — Строишь структуру отчёта (4 листа): Параметры → Денежные потоки → ИТОГ → Чувствительность.

**ШАГ 3** — Описываешь формулы как бизнес-правила (см. список выше).

**ШАГ 4** — Передаёшь это в MCP Excel Server через `/generate-report <name>`. Скилл `/generate-report` — это **переводчик** бизнес-правил в конкретные MCP-вызовы (он знает, в какие ячейки какая формула попадает). Читает `projects/<name>.json`, создаёт `reports/<name>_finmodel.xlsx`.

## 🛠 Скиллы

| Команда | Что делает |
|---|---|
| `/new-project` | Диалог: собрать входные параметры → `projects/<name>.json` |
| `/generate-report <name>` | **Основная команда.** Excel-отчёт через MCP. Включает листы Параметры/Денежные потоки/ИТОГ + **автоматически** заполняет Чувствительность |
| `/sensitivity <name>` | (опционально) Перезаписать лист «Чувствительность» с другими сценариями. По умолчанию заполняется автоматически в `/generate-report` |
| `/analyze <name>` | Прочитать KPI через MCP → бизнес-заключение |
| `/load-excel <путь>` | Загрузить параметры из существующего Excel |

Типовой сценарий: `/new-project` → `/generate-report <name>` → `/analyze <name>`.
Чувствительность отдельной командой запускать **не нужно** — она встроена в `/generate-report`.

## 📚 Технический справочник (для скиллов, не для общения с пользователем)

Скиллы используют **два MCP-сервера**: основной `finmodel` (наш собственный) и
запасной `excel` (низкоуровневый). Пользователю об этом не рассказывать.

### MCP-сервер `finmodel` — основной

Наш собственный сервер: `finmodel/mcp_server.py`. Запускается через `python -m finmodel.mcp_server`.

| Инструмент | Назначение |
|---|---|
| `mcp__finmodel__list_projects` | Список проектов в `projects/` |
| `mcp__finmodel__get_project_params(name)` | Сырой JSON проекта |
| `mcp__finmodel__get_kpi(name)` | NPV/IRR/ROI/Payback без построения Excel |
| `mcp__finmodel__build_project(name)` | **Полный отчёт** + KPI одним вызовом |
| `mcp__finmodel__compare_projects([a,b,c])` | Сравнение KPI нескольких проектов |

`build_project` строит файл `reports/<name>_finmodel.xlsx` со всеми формулами,
4 листами и анализом чувствительности. **Один вызов = готовый отчёт.**

### MCP-сервер `excel` — запасной

`uvx excel-mcp-server stdio`, `EXCEL_FILES_PATH` = корневая папка проекта (рабочая директория сессии).

Используется только если нужно **читать** или **точечно править** существующий
отчёт (например, открыть готовый Excel и прочитать конкретную ячейку для `/analyze`).
**Не использовать для построения модели** — для этого есть `finmodel`.
Пути в `filepath` — относительно `EXCEL_FILES_PATH`.

### Используемые MCP-инструменты

| Инструмент | Назначение |
|---|---|
| `mcp__excel__create_workbook(filepath)` | Создать файл |
| `mcp__excel__create_worksheet(filepath, sheet_name)` | Добавить лист |
| `mcp__excel__delete_worksheet(filepath, sheet_name)` | Удалить лист |
| `mcp__excel__get_workbook_metadata(filepath)` | Список листов |
| `mcp__excel__write_data_to_excel(filepath, sheet_name, data: List[List], start_cell)` | Записать 2D-массив (числа/строки/формулы) |
| `mcp__excel__apply_formula(filepath, sheet_name, cell, formula)` | Одна формула с верификацией |
| `mcp__excel__copy_range(filepath, sheet_name, source_start, source_end, target_start)` | Размножить шаблонную строку по диапазону |
| `mcp__excel__read_data_from_excel(filepath, sheet_name, start_cell, end_cell)` | Прочитать диапазон |
| `mcp__excel__format_range(filepath, sheet_name, start_cell, end_cell, ...)` | Форматирование |

### Принцип построения модели через MCP

1. **Один шаблон-строка на фазу.** Записывается одна строка формул со ссылками — Excel сам сдвинет относительные ссылки при `copy_range`.
2. **Никогда не писать N×M формул в цикле.** Это медленно и ломается на больших горизонтах.
3. **Все KPI на листе ИТОГ — формулы Excel** (NPV, IRR, MATCH, SUM, SUMIF).
4. **Сценарии чувствительности — формулы-ссылки** вида `='Параметры'!$C$5*1.2`, не предвычисленные значения.

Детали соответствия «бизнес-правило → ячейка» хранятся в `.claude/skills/use-template.md`. Агент не показывает их пользователю.
